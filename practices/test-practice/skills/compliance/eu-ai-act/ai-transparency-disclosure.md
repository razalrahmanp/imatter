# ai-transparency-disclosure — AI Transparency Disclosure Pattern

## Pattern Summary

Limited-risk AI systems must disclose AI involvement to users before interaction. Disclosure must be clear, prominent, and not buried in terms of service.

**Disclosure requirements by system type (Art. 50):**

```
Chatbots / conversational AI:
  → Inform user they are interacting with an AI — before or at start of interaction
  → Exception: obvious from context (e.g. clearly labelled robot assistant)

AI-generated text / images / audio / video (deepfakes):
  → Label as AI-generated in machine-readable format AND visible to user
  → Exception: artistic/creative work where disclosure would impair the work (narrow)
  → Exception: authorised law enforcement use

Emotion recognition systems:
  → Inform individuals whose emotion is being recognised — before inference
```

**Chatbot disclosure component (React):**
```tsx
function AiDisclosureBanner() {
  return (
    <div role="alert" aria-live="polite"
         className="bg-navy-800 border border-navy-700 rounded p-3 text-navy-300 text-sm font-sans">
      This response was generated by an AI assistant. It may contain errors.
      Always verify financial or operational decisions with a human.
    </div>
  );
}
// Render at the top of any AI-generated content block, not just on first load
```

**API response — machine-readable label:**
```typescript
// Include in every AI-generated API response
interface AiGeneratedResponse<T> {
  data: T;
  _ai_generated: true;
  _model: string;       // e.g. "amazon.nova-pro-v1:0"
  _generated_at: string;
}
```

## Full Reference

### What counts as "obvious from context"
Regulator guidance: a clearly named "AI Chat" widget with a robot icon may qualify. An assistant named after a human (e.g. "Sarah from Support") does not qualify — must disclose explicitly.

### Disclosure placement rules
- Must appear before or at the start of interaction — not discoverable only via a settings page
- Must be in the same language as the interface
- Must be accessible (WCAG 2.1 AA — sufficient contrast, screen reader compatible)

### Forbidden
- Launching a chatbot without per-session disclosure when context is ambiguous
- Hiding disclosure inside a collapsed "About" section or footer link
- Using machine-readable AI labels as the only disclosure (human-readable also required)
